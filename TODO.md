
The load ski function is not populating data on other tabs when a ski definition json file is opened. Loading a file from any tab should populate all possible data in all tabs.

Base design: The ski outline visualisation is appearing as a ski centreline. The base outline does not show the inward steps for the cutout to accomodate the edges: see ./data/base_tip_steps.png which shows the tip section of a base outline for an example of what it should look like. Add a feature to generate G code to cut this base out of plastic sheet material. There should be options to use a CNC router, or a CNC drag knife. Add field to enter the necessary cutting settings / tool details.

Camber design: Remove orange dashed line showing Rocker/camber boundary. The camber line definition is not working well. The camber line should consist of 3 splines. The first connects the tip to the tip rocker / camber junction. At this junction the control arm should be constrained to be horizontal with the junciton point so the spline is horizontal at that point. A second spline defines the camber section, starting at the tip rocker junction, passing through the max camber point (through a control point that constrains the spline to be horizontal at the max camber point), and ending at the tail rocker junciton. Again plot the control points and allow them to be dragged horizontally in the visualisation. This should look and work very similarly to the system in the Outline design tab. The third section for the tail rocker is similar to the tip section. ./data/camber_line.png shows an example of the geometry and where the spline control points should be, although it is for a mould seciton so shows some extra strait lines that are irrelevent.
